- **Voice Tutor — added two complementary knobs for slowing conversation pace** (commit `52258e7`, branch `feat/study-companion-mode`). Discussion started from the question "is TTS_SPEED the best way to control conversation speed?" and surfaced that "speed" has several meanings: TTS rendering speed, VAD stop seconds, response brevity, and end-to-end latency. Landed on stacking two levers: (1) `VOICE_TUTOR_TTS_SPEED` env flag (`bot.py:186`) — wraps Cartesia Sonic-3's `GenerationConfig(speed=...)`, valid range 0.6–1.5, unset → baseline behavior preserved. Sonic-3 generates at the target speed natively (model conditioned on speed, not post-hoc time-stretching), so moderate values like 0.85–0.9 stay clean; extremes near 0.6 can produce drawn-out vowels. (2) Prompt-level cadence cue appended to `BREVITY_REMINDER` (`bot.py:307`): "Speak deliberately — use commas and brief pauses; don't rush." Sonic-3 honors punctuation as prosody, so more commas/em-dashes from the LLM yields natural pauses at zero quality cost. Prompt cue is the primary lever (free, no rendering artifacts), TTS speed flag is the backstop if cadence alone isn't enough. Both kept in despite the user not hearing a dramatic difference in one test session — both are cheap-to-revert (env flag default None, prompt edit is one line) so leaving them in costs nothing.
