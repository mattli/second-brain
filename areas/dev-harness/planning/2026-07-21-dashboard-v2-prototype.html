Goal

Build a read-only local web dashboard for the active or most recent dev-harness run.

What it does

Serves a single page showing the live state of a run, rendering only data that already exists in the run folder. The page updates in place via polling — a continuously live view with no visible reload — and never writes to the run folder or any config.

What it shows

  • The goal, and the current sprint / round / step
  • Per-sprint scores
  • Elapsed wall-clock time
  • Spend, when it's recorded

Constraints

  • Stdlib-only server — no new dependencies
  • Strictly read-only — never writes to the run folder or any config
  • Degrades gracefully when fields are missing or a run is mid-write

Out of scope (stage 2)

  • True push streaming (SSE / WebSocket)
  • Launch / stop controls and config editing
