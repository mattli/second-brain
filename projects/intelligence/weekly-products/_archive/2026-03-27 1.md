# Product Briefing — 2026-03-27

## Webb — Document Intelligence Platform
*Who has this problem?* Investigative journalists, researchers, lawyers, and curious citizens trying to navigate massive public document dumps (Epstein files, FBI Vault, 9/11 archives).
*What problem does it attempt to solve?* Millions of pages of public documents are technically accessible but practically unsearchable. Webb indexes them with hybrid BM25 + semantic search, entity resolution, and a visual investigation canvas — at $20/mo instead of the $500K enterprise stack it replaces.
*Is the solution complete, or does it address only part of the pain?* Strong on search and entity graph. The value compounds as more datasets are added — currently limited to public archives; private document upload isn't the focus yet.
*What would a better solution look like?* User-uploaded private document sets with the same pipeline, plus collaborative investigation workspaces.

## Starpod — Open-Source AI Agent Runtime
*Who has this problem?* Developers building AI agents that need to serve multiple users or clients without data leaking between tenants.
*What problem does it attempt to solve?* Running the same AI agent for multiple users typically means shared memory and context — a real isolation problem. Starpod (Rust, MIT) lets you define an agent once and deploy isolated instances per user, each with its own memory, encrypted vault, filesystem, and sessions. Multi-channel out of the box: Telegram, Slack, web UI, HTTP API.
*Is the solution complete, or does it address only part of the pain?* Core runtime is solid (16 crates, 1,316 tests, v0.1.6 shipped Mar 26). Managed cloud console is early. The self-extending skills feature (agent creates/edits its own skill files at runtime) is a genuinely novel primitive.
*What would a better solution look like?* Richer agent-to-agent communication patterns and a marketplace for reusable skill libraries.

## Seevora — Homepage Clarity Scanner
*Who has this problem?* Founders, SaaS teams, and agencies who ship polished homepages that still confuse visitors — the copy sounds professional but the product category and audience don't land immediately.
*What problem does it attempt to solve?* Paste a URL, get a structured report across four dimensions: clarity, discoverability, AI visibility, and persuasion — with prioritized fixes. Also ships an MCP server for Cursor and Claude Desktop so developers can scan unpublished local builds before deploying.
*Is the solution complete, or does it address only part of the pain?* Addresses the diagnostic layer well. Doesn't auto-rewrite copy — intentionally. The MCP integration is a smart distribution move.
*What would a better solution look like?* Continuous monitoring with diff-based alerts when messaging scores degrade after copy changes.

## SnapMonk — App Store Screenshot Generator
*Who has this problem?* Indie developers who need professional-looking App Store and Google Play screenshots but won't pay $50+/mo for design tools.
*What problem does it attempt to solve?* 25+ device frames, templates, backgrounds, one-click export at every required dimension, 50-language localization — free tier with watermark, $19/mo without. Used by 2,500+ developers, no account required to start.
*Is the solution complete, or does it address only part of the pain?* Covers the screenshot creation problem end-to-end. Doesn't address ASO copy or A/B testing within the tool itself.
*What would a better solution look like?* Integrated A/B test tracking against App Store conversion data and AI-generated screenshot copy suggestions.

## TrailKast — Self-Hosted SSH Audit Trail
*Who has this problem?* Engineering teams that need SOC 2, HIPAA, PCI DSS, or ISO 27001 compliance but currently have zero visibility into what engineers actually do in production SSH sessions.
*What problem does it attempt to solve?* One-command install on Linux servers; records every SSH session at the PTY level including activity inside Docker containers and Kubernetes pods. Full terminal replay. Data never leaves your server. Currently on waitlist.
*Is the solution complete, or does it address only part of the pain?* Strong on capture and replay. Compliance report generation (audit-ready exports for specific frameworks) appears to be on the roadmap but not yet shipped.
*What would a better solution look like?* Anomaly detection on command patterns (e.g., flag when a dev accesses a secrets directory) with Slack/PagerDuty alerting.

## Today's Signal
Builders today are solving problems at the intersection of compliance, isolation, and trust — SSH audit trails, per-tenant AI agent isolation, private finance tracking. The other cluster is "surface polish infrastructure": screenshot generators, homepage scanners, brand strategy tools for solo founders. The market is telling builders that both getting something built and making it legible to others remain genuinely unsolved, even in 2026.
