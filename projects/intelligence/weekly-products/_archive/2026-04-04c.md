## Problem Categories

### AI Coding Infrastructure: Memory, Orchestration, and Cross-Model Integration

- **Who it's for:** Developers already using Claude Code or Codex who want to extend those tools — add persistent memory, structured workflows, or cross-model delegation — without switching environments.
- **Problem it's solving:** AI coding assistants reset context with every session. You explain the codebase once; the next day, you explain it again. A parallel problem: Codex and Claude are competing CLIs with different strengths, but there's no clean way to use both in the same workflow. Builders are now shipping the missing infrastructure layer for both.
- **What's being built:** A persistent memory substrate for Claude Code, a workflow orchestration layer for Codex CLI (mirroring what oh-my-claudecode did for Claude), and an official cross-model plugin from OpenAI that lets Claude Code users delegate tasks directly to Codex. All three shipped or reached significant traction this week.
  - [ekkOS](https://ekkos.dev) — Persistent memory for AI coding assistants: gives Claude Code a real brain that compounds across sessions. Modules include Cortex (memory), Pulse, Nerve, and Synk. Free tier, 30-second deploy. 3,000+ hours invested, launched on Product Hunt this week. (@ekkostech)
  - [oh-my-codex](https://github.com/Yeachan-Heo/oh-my-codex) — Workflow orchestration layer for OpenAI Codex CLI by the same developer as oh-my-claudecode (10.4K stars). Adds autopilot mode, agent teams, HUDs, skill reuse, and structured planning/verification pipelines. 9.2K+ GitHub stars. (@Yeachan-Heo / @bytecrafter_1, ~11K stars cited on April 3)
  - [OpenAI Codex Plugin for Claude Code](https://github.com/openai/codex-plugin-cc) — Official Apache 2.0 plugin from OpenAI staff: lets Claude Code users invoke Codex for code review, adversarial review, or delegated execution directly inside Claude Code. Delegates through the local Codex CLI and app server. Launched March 30. (@reach_vb, 360 reposts on announcement tweet)
- **Engagement signal:** The @reach_vb announcement of the OpenAI plugin was the week's highest-signal cross-company developer post: 360 reposts, rare for a pure tooling announcement. The "oh-my" pattern (oh-my-claudecode, oh-my-codex) has become a real category: same orchestration surface repackaged for each AI CLI as it gains adoption. ekkOS addresses the persistent pain of session resets — the most common friction point cited in Claude Code forum threads — and is the only standalone company (not a solo script) targeting this problem as a product.

## Summary Observations

The most structurally significant event of the week was OpenAI shipping an official plugin for Claude Code. Two competing platforms have become interoperable at the plugin layer. That one of them submitted to the other's plugin architecture — rather than building a walled garden — suggests the developer tooling ecosystem is settling into a platform-and-plugin model where the underlying CLIs are less important than the orchestration layer above them.

The "oh-my" pattern (oh-my-claudecode → oh-my-codex) is the week's most useful signal for tracking developer tooling adoption curves. When a developer ships the same orchestration framework for a new CLI within days of that CLI gaining traction, it signals that the underlying CLI has hit an adoption threshold worth instrumenting. oh-my-codex reaching 9.2K stars within its first weeks confirms that Codex CLI adoption is now real, not experimental.

ekkOS is the week's most technically substantive product from an independent builder: 3,000+ hours of work on a persistent memory substrate specifically designed for Claude Code, launched on Product Hunt this week. The problem it solves (session context resets) is the most consistent complaint in Claude Code forums. It competes with Anthropic's own Claude Import Memory feature (launched this week on Product Hunt by Anthropic), which transfers context from other AI providers into Claude's native memory. The market for "AI context persistence" is now contested between indie tooling and the platform itself.

MCP protocol shipped OAuth 2.1, streamable HTTP, and JSON-RPC batching in one update this week (@_SajjadHasan_, April 1). The security layer was the missing piece for enterprise MCP adoption — now present. Separately, Runway (the AI video company) launched a $10M venture fund and Builders program for early-stage AI, media, and world simulation startups. These are the week's two largest institutional signals for the builder ecosystem: MCP becoming production-grade infrastructure, and a major AI company now explicitly funding the next generation of AI tool builders.
