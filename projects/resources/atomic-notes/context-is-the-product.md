Every AI product I've looked at seriously is "just prompting" with good context. Cursor, Claude Code plugins, my own briefing system — they're all thin wrappers around LLM calls with smart context assembly. The ones that win nail the UX and make context assembly invisible.

For pmtxt, the structured context files (customer personas, stakeholders, revenue model, competitors) are the actual product. The AI call is cheap. The value is in how the context is assembled, maintained, and applied. Context compounds over time — every decision logged, every input, every piece of feedback makes the next recommendation sharper. The knowledge base is the moat.

This applies to my own Second Brain too. The system's value isn't in any individual feature — not the briefing, not the to-do list, not the wiki. It's in the accumulated context across all of them. The agent knows what I'm working on, what I've read, what I've been circling, what stresses me out. That composite understanding is what makes it useful, and it's something no off-the-shelf tool can replicate because it's built from my specific decisions over time.

The "just prompting" concern dissolves when you realize that assembling and maintaining the right context is the hard part, not the prompting itself.
