The future of AI products is outcome-based. Not tools that help you do things, but services that deliver results. Autopilot, not co-pilot.

Cursor's outcome isn't "code files" — it's "working software, faster." Code is a side effect. Similarly, if I built a PM tool, the outcome shouldn't be a PRD — it should be confidence that you're building the right thing, with the PRD as a side effect.

This framing passed my obsolescence test: apply "what if GPT-6 launches tomorrow" to the OUTCOME, not the FEATURE. Features get commoditized. Outcomes don't. If you sell "write a PRD," a better model can do that for free. If you sell "know what to build next and be right about it," that's durable because the value is in the accumulated context and the judgment loop, not the generation.

The tension: PM decisions have slow feedback loops. Weeks or months to know if a feature worked, unlike code that runs or doesn't immediately. That makes it harder to prove value quickly. But it also means the compounding effect of context is even more valuable — if the system gets better at predicting which bets pay off over time, that's a genuine moat.
