### AI Agent Execution, Safety & Governance

- **Who it's for:** Developers building production AI agents, enterprise teams deploying agentic workflows, and infrastructure engineers evaluating where agents can go wrong
- **Problem it's solving:** AI agents can now execute real actions — write code, make API calls, transfer funds, manage tasks — but the control layer around those actions is almost entirely absent. Builders have no standard way to define operational boundaries, verify that outputs are grounded in real sources, score whether their infrastructure can even serve agents correctly, or handle economic settlements between agent-to-agent transactions.
- **What's being built:** Graph-based agent execution controllers, long-term decentralized AI memory, RAG provenance middleware, AI ecosystem intelligence APIs, website agent-readiness scorers, blast radius risk calculators, agentic project management platforms, and escrow infrastructure for AI-to-AI payments.
  - [Switchplane](https://t.co/6kAhx1OfHx) — deterministic control for agent execution; the graph defines operational boundaries; subprocess isolation so each agent runs as a separate process; local-first, no containers, no managed services (@demianbrecht, 1 like)
  - [ShellBrain](https://t.co/psyCd3sKnf) — decentralized long-term memory for AI engineering tasks; four memory types (procedural, semantic, episodic, working); problems stored only as structured cases with outcomes, never in isolation; case-based reasoning (@shellbrain_ai, 0 likes)
  - [Dokis](https://t.co/s6lToV6HQU) — lightweight RAG provenance middleware; verifies every claim in an LLM response is grounded in a retrieved source without making an LLM call; v0.3.0, open source on GitHub; config supports stale-source detection and guardrail enforcement mode (@Vbj0818, 0 likes)
  - [Strata](https://t.co/D1bBKmfiqG) — verified knowledge API built for agents; native MCP server; connect in one command and the agent gets verified best practices and latest ecosystem data; 100 calls/month free, $29/mo Pro with daily index refresh (@hear_pt, 0 likes)
  - [COVEN AI](https://t.co/mD89V39jrk) — scores any website's agent readiness on five dimensions (Structure, Citability, Discoverability, Agent Interface, Transactability); free scan, no account required; cites Cloudflare data showing only 4% of the top 200,000 domains have any Content Signal directives for AI agents (@coven0x, 0 likes)
  - [Cycles](https://t.co/P1FuOjSk0B) — AI agent blast radius risk calculator; name the agent, model the actions, dial containment percentage, see the monthly blast radius estimate; state encoded in URL for sharing and audit trails (@amavashev, 1 like)
  - [Formilist AI](https://t.co/5gq207go2i) — B2B agentic task and project management platform for enterprise teams; positioned as "most PM tools tell you what's happening — Formilist makes things happen"; AI agents assign, execute, and track tasks autonomously (@formilistai, 0 likes)
  - [OnlyTrust](https://t.co/wQR0rYkqCq) — escrow, checks, and settlement for AI agent work; AI agents can publish services and be paid by other AI agents after work is verified; smart contracts on Base Sepolia; registry and docs published (@OnlytrustAI, 0 likes)

OnlyTrust is the earliest-stage product in this category, relying on L2 testnet infrastructure. ### Tools for Building With Claude & AI More Efficiently

- **Who it's for:** Individual developers, solo founders, and builders actively using Claude Code, Claude Projects, or any LLM-based coding workflow daily
- **Problem it's solving:** The workflow around AI coding tools is still rough: prompts lack context, Claude Projects don't know your preferences without a lengthy setup, AI spend leaks silently into unexpected costs, and building a real phone-call AI agent requires manual SIP/STT/TTS wiring that has nothing to do with the AI logic itself.
- **What's being built:** Skill file generators that make Claude work like you, CLI prompt inflaters that load context from fragments, macOS rate-limit trackers that read Claude Code sessions locally, and open-source telephony frameworks that abstract SIP away from AI conversation logic.
  - [Relatch](https://t.co/FMIH6GyHI6) — generates a Claude skill file from your existing files and data; one `.md` file you drag into Claude Projects; Claude then writes in your voice and follows your rules without being told twice; all processing runs in-browser, files never touch a server (@harit_manas, 0 likes)
  - [inflate](https://t.co/IGmXEHeSl8) — context-aware CLI prompt inflater for Claude Code; type a short fragment, get a fully context-loaded prompt ready to paste; open source on GitHub (@Joncik91, 0 likes)
  - [Tokens 4 Breakfast](https://t.co/CnWsrqW2G6) — macOS AI spend guard; reads Claude Code session files locally (extracts only usage metadata, never message content); rate limit warning before sessions get cut; free tier for 1 provider, one-time Pro; built in Germany, runs entirely on-device (@onekapisch, 1 like)
  - [SIPHON](https://t.co/MsgcOy8jCE) — open-source Python framework for AI telephony agents; handles SIP, real-time audio, STT, LLM, and TTS orchestration so you only write conversation logic; vendor-agnostic (swap Anthropic/OpenAI, Deepgram/AssemblyAI with config change); PSTN-first with Twilio/Telnyx native integration; zero-config horizontal scaling (@blackdwarf__, 0 likes)
- **Engagement signal:** Tokens 4 Breakfast had the most polished product page in this category. Relatch and inflate are both solo-founder tools aimed squarely at the Claude Code workflow, which has emerged as its own micro-ecosystem. SIPHON is the furthest from Claude specifically but addresses the gap where AI conversation agents need real telephony infrastructure. Thread context: @MicroLaunchHQ (202 replies, 86 likes), @RoundtableSpace.

## Summary Observations

The dominant theme this week is the hardening of AI agent infrastructure. Eight products landed in a single briefing all attacking different parts of the same problem: Switchplane governs what agents can do, ShellBrain gives them long-term memory, Dokis verifies their outputs are grounded, Strata feeds them verified knowledge, COVEN AI scores whether the web is ready to serve them, Cycles quantifies the damage when they go wrong, Formilist lets them manage tasks autonomously, and OnlyTrust settles payments between them. The category is no longer just "agent memory" (covered in the April 25 briefing) or "agent safety" (April 18) — it is now filling in every layer of the agent execution stack from prompt to payment. The density and diversity of this week's crop suggests the category is entering a rapid expansion phase.

The second notable pattern is a continued builder focus on privacy-first design as a standalone value prop, not just a compliance requirement. Tokens 4 Breakfast explicitly calls out "the actual message content, your prompts, and AI responses are never read" — and proves it by describing the exact code path. Monetor leads with "no bank login." BankConvert runs parsing locally. dAIly processes messages in-memory and never writes them to disk. Relatch runs all processing in the browser. This is now appearing across categories — consumer apps, developer tools, and finance — consistently enough to be considered a positioning strategy rather than a feature.

SoilIQ 2.0 (3 likes) was the highest-engagement product of the week by a clear margin, and it's a native iOS app about soil temperature. Its success in a builder thread dominated by SaaS products underlines a recurring signal in these briefings: deep domain specificity — tools that know exactly one problem and know it better than anyone — consistently outperform broad-scope tools in engagement, even in developer communities where you'd expect developer tools to dominate.

The Indie GTM cluster (spark, iaco, MakeMyDemo) reflects a continued market filling around the zero-to-first-user phase of solo SaaS. The convergence of landing-page testers, distribution courses, and demo video generators around the same stage of the builder journey suggests the "after you build, before you have users" moment is now recognized as a distinct and underserved product category of its own.

Finally, the writing and publishing cluster (Inkett, makeEbook, repod, Vvd) is the most ambitious category in this week's briefing by scope. Inkett in particular is positioning as the entire publishing stack for working novelists — craft + business + AI in one platform. The question for all four of these products is whether the writing-for-income audience (which historically accepts friction from specialist tools) is large enough to sustain a product with this level of specificity.
