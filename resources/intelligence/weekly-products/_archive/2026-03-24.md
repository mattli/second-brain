# Product Briefing — 2026-03-24

## everything-claude-code
**Who has this problem?** Developers using Claude Code, Cursor, Codex, or OpenCode who want more structure, reliability, and performance from their AI coding sessions
**What problem does it attempt to solve?** A community-built plugin repo that ships 28 agents, 119 skills, 60 commands, rules, hooks, and MCP configurations as a unified harness for Claude Code (and now Cursor/Codex/OpenCode). It handles the scaffolding work most devs do manually — session memory, TDD workflows, security review, language-specific patterns — and makes it installable in one command. Hit 4,453 new GitHub stars today and 103k total.
**Is the solution complete, or does it address only part of the pain?** Well-maintained and surprisingly comprehensive across 10 languages with 992 internal tests. The real gap is that it still requires manual rules installation (plugin system can't distribute them automatically), and heavy configs can slow down context windows for simple tasks.
**What would a better solution look like?** A native agent harness layer built into the coding agent itself — where skills, memory, and rules are dynamically loaded based on the task at hand rather than always-on. The community is essentially building what the tools should ship by default.

## cq — Stack Overflow for AI Agents (Mozilla AI)
**Who has this problem?** Teams running AI coding agents that repeatedly hit the same walls — wrong API assumptions, CI failures, stale documentation — and burn tokens solving the same problems in isolation
**What problem does it attempt to solve?** An open-source shared knowledge commons where agents can query solved problems before they start work and contribute new learnings back. Before touching an API integration a Claude Code agent hasn't seen, it checks cq to see if another agent already discovered that Stripe returns 200 with an error body on rate limits. Knowledge earns trust through repeated use across agents. Mozilla AI shipped a working PoC with plugins for Claude Code and OpenCode, an MCP server, team API, and UI for human review.
**Is the solution complete, or does it address only part of the pain?** Very early — the PoC works but the confidence scoring, reputation, and staleness-detection systems that would make it trustworthy at scale are still being designed. The harder problem is canonicalization: how does knowledge from one codebase generalize to another without hallucinating false patterns?
**What would a better solution look like?** cq itself describes the destination well — trust signals that emerge through use, not authority. The missing layer is automatic staleness detection (API versioning, library updates) and structured contribution incentives so agents don't contribute low-quality noise when they're uncertain.

## My Computer by Manus AI (Meta)
**Who has this problem?** Power users and non-technical people who want an AI agent to handle tedious local computer tasks — file organization, batch renaming, app building — without needing to write scripts or open a terminal
**What problem does it attempt to solve?** Manus (acquired by Meta) expanded its cloud-only agent to run directly on Mac and Windows via a desktop app. Through CLI execution in the local terminal, the agent can organize folders, rename files, build local apps, use the machine's GPU, and orchestrate tasks remotely from a phone. It bridges cloud intelligence (Gmail/Calendar integrations) with local compute. Claimed PH #1 today with 255 upvotes.
**Is the solution complete, or does it address only part of the pain?** The file management and scripting demos are polished, but the real capability is still developer-oriented — CLI-based execution means errors surface as terminal output that non-technical users can't debug. The "allow once / always allow" trust model is the right direction but approval fatigue will hit quickly for complex tasks.
**What would a better solution look like?** Full computer-use (GUI interaction, not just CLI) combined with a recovery model where the agent can self-diagnose and roll back failed operations. The local-first direction is right; OpenClaw already occupies this space for developers — Manus is betting it can do it for everyone else.

## Lightning Rod — Training Data from the News
**Who has this problem?** ML engineers and AI product teams who need labeled training data but face the bottleneck of manual annotation or synthetic data that drifts from real-world language
**What problem does it attempt to solve?** Generates labeled training datasets automatically from live public news sources — no manual labeling required. Real-world news provides grounded, current-language examples for classification, NER, and fine-tuning tasks. Launched on PH with 193 upvotes today (relaunched from its March 17 debut).
**Is the solution complete, or does it address only part of the pain?** Strong for NLP and classification tasks anchored to current events, weak for anything requiring private, domain-specific, or non-English data. News bias and recency windows also limit generalizability for tasks that need long-horizon or specialized corpora.
**What would a better solution look like?** A generalized real-world data harvesting layer that pulls from a configurable set of sources (not just news), with deduplication, label-quality scoring, and direct integration into fine-tuning pipelines. The gap between "data collection" and "production-ready training set" is still wide.

## Today's Signal
The meta-layer for AI development is being built in public at speed: a community plugin harness for coding agents, a shared knowledge commons so agents stop solving the same problems twice, a desktop bridge to local compute, and automated training data pipelines. The pattern is infrastructure — builders are no longer just shipping AI features, they're building the scaffolding that makes AI development itself faster and more reliable.
